# Telegram VPN Bot для Marzban Panel

Телеграм бот для управления VPN пользователями через Marzban Panel.

## Возможности

- 🔐 Создание и удаление VPN пользователей
- 📊 Получение статистики использования
- 💳 Интеграция с платежными системами
- 🎫 Управление подписками
- 📱 QR-коды для быстрого подключения
- 👨‍💼 Админ панель для управления

## Требования

- Python 3.8+
- Marzban Panel (установлен и настроен)
- Telegram Bot Token
- Доступ к Marzban API

## Установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd marzban-telegram-bot
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Скопируйте файл конфигурации:
```bash
cp .env.example .env
```

4. Отредактируйте `.env` файл:
```bash
nano .env
```

### Настройка переменных окружения

#### Обязательные параметры:

- `BOT_TOKEN` - Токен вашего Telegram бота (получить у @BotFather)
- `MARZBAN_API_URL` - URL вашей Marzban панели (например: https://panel.example.com)
- `MARZBAN_USERNAME` - Имя пользователя администратора Marzban
- `MARZBAN_PASSWORD` - Пароль администратора Marzban
- `ADMIN_ID` - Ваш Telegram ID для доступа к админ функциям

#### Опциональные параметры:

- `DATABASE_URL` - URL базы данных (по умолчанию SQLite)
- Payment API ключи для интеграции с платежными системами
- Webhook настройки для продакшн развертывания

## Настройка Marzban Panel

1. Убедитесь что Marzban Panel установлен и работает
2. Включите API доступ в настройках панели
3. Создайте административного пользователя или используйте существующего
4. Проверьте доступность API по адресу: `https://your-panel.com/docs`

### Структура API Marzban

Бот использует следующие endpoints:

- `POST /api/admin/token` - Аутентификация
- `GET /api/users` - Список пользователей  
- `POST /api/user` - Создание пользователя
- `GET /api/user/{username}` - Информация о пользователе
- `DELETE /api/user/{username}` - Удаление пользователя
- `GET /sub/{username}` - Subscription URL пользователя

## Запуск

1. Тестовый запуск:
```bash
python bot.py
```

2. Проверьте подключение к API:
Отправьте `/test_api` боту (только для админа)

3. Для продакшн развертывания используйте:
```bash
# С использованием systemd
sudo systemctl start marzban-bot

# Или с использованием screen/tmux
screen -S marzban-bot python bot.py
```

## Основные команды бота

- `/start` - Главное меню
- `/test_api` - Тест подключения к API (только админ)
- Кнопки в меню для управления подписками

## Миграция с других панелей

Если вы переходите с другой панели (например, 3x-ui, RemnaWave), основные изменения:

1. **API структура**: Marzban использует стандартную REST API структуру
2. **Аутентификация**: Через username/password вместо API ключей
3. **Пользователи**: Создаются через `/api/user` endpoint
4. **Конфигурации**: Доступны через `/sub/{username}` URL

## Устранение неполадок

### Ошибки аутентификации
- Проверьте правильность MARZBAN_USERNAME и MARZBAN_PASSWORD
- Убедитесь что пользователь имеет административные права

### Ошибки API подключения  
- Проверьте доступность MARZBAN_API_URL
- Убедитесь что Marzban Panel запущен
- Проверьте настройки SSL/TLS

### Ошибки создания пользователей
- Проверьте квоты и лимиты в Marzban
- Убедитесь что протоколы настроены правильно

## Поддержка

При возникновении проблем:
1. Проверьте логи бота
2. Используйте команду `/test_api` для диагностики
3. Проверьте статус Marzban Panel
